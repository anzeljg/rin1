7.

Prepiši sledeči algoritem iz psevdokode v pythonov program:


Kakšen problem rešuje ta algoritem?

Algoritem rešuje problem iskanja največjega skupnega delitelja dveh pozitivnih celih števil.

# Vrne največji skupni delitelj pozitivnih
# celih števil a in b.
def gcd(a, b):
while a != b:
if a > b:
a -= b
else:
b -= a
return a

8.

Problem iskanja največjega skupnega delitelja lahko rešimo tudi z (v splošnem počasnejšim) algoritmom, ki novo spremenljivko (npr. d) najprej postavi na manjše od obeh števil, nato pa jo zmanjšuje tako dolgo, dokler obe vhodni števili ne postaneta deljivi z d. Zapiši algoritem v obliki pythonove funkcije.

# Vrne največji skupni delitelj pozitivnih
# celih števil a in b (alternativni algoritem).
def gcd2(a, b):
if a < b:
d = a
else:
d = b

# Spremenljivko d zmanjšuj, dokler nista
# z njo deljivi obe števili.
while a % d != 0 or b % d != 0:
d -= 1
return d

9.

Pokaži, da oba algoritma za računanje največjega skupnega delitelja proizvedeta enak izhod za vse pare celih števil a ∈ {1, . . . , 100} in b ∈ {1, . . . , 100}.

Napiši in preizkusi pythonovo kodo, ki za vsak par števil a in b pokliče obe funkciji in preveri ujemanje rezultatov.

# Prvi algoritem za računanje največjega
# skupnega delitelja.
def gcd(a, b):
while a != b:
if a > b:
a -= b
else:
b -= a
return a

# Drugi algoritem za računanje največjega
# skupnega delitelja.
def gcd2(a, b):
if a < b:
d = a
else:
d = b
while a % d != 0 or b % d != 0:
d -= 1
return d

# Koda za preverjanje delovanja obeh algoritmov.

# Poženemo obe funkciji za vse pare celih
# števil od 1 do vključno 100 in preštejemo,
# pri koliko parih se obe funkciji ujemata.
stEnakih = 0
stRazlicnih = 0

for a in range(1, 101):
for b in range(1, 101):
prvi = gcd(a, b)
drugi = gcd2(a, b)
if prvi == drugi:
stEnakih += 1
else:
stRazlicnih += 1

print('Št. enakih rezultatov:', stEnakih)
print('Št. različnih rezultatov:', stRazlicnih)

10.

Utemelji, zakaj sledeči algoritem pravilno izračuna število števk podanega pozitivnega celega števila.
def steviloStevk(n):
p = n
s = 0
while p > 0:
p = p // 10
s += 1
return s

Izraz p = p // 10 številu p odbije zadnjo števko. To pomeni, da se zanka izvrši tolikokrat, kot znaša število števk števila n. Ker ima spremenljivka s na začetku vrednost 0, v vsakem obhodu zanke pa se poveča za 1, je njena vrednost po koncu izvajanja zanke enaka številu števk števila n.

11.

Dokaži, da se algoritem iz prejšnje naloge ustavi pri vseh pozitivnih celih številih n.

Algoritem se ustavi, ker je število p na začetku vsakega obhoda zanke pozitivno, v vsakem obhodu zanke pa se zmanjša najmanj za 1, saj izgubi zadnjo števko.

12.

Ali se sledeči algoritem ustavi pri vseh pozitivnih celih številih n? Kaj pa, če bi zančni pogoj spremenili v n > 1?
def zanka(n):
while n > 0:
n = (n + 1) // 2

Odgovor na prvo vprašanje je negativen, saj v neskončno zanko zabredemo že v primeru, če ima parameter n vrednost 1.

Če bi zančni pogoj spremenili v n > 1, pa bi se zanka ustavila pri vseh pozitivnih celih številih n, saj je vrednost spremenljivke n vedno pozitivno celo število, obenem pa se v vsakem obhodu zanke zmanjša najmanj za 1. Vrednost izraza (n + 1) // 2 je namreč manjša od n za vsak n > 1.
