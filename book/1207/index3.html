Namig

Pomagaj si s seštevanjem: na primer, 3 · 4 lahko izračunaš kot 3 + 3 + 3 + 3.

Rešitev

# Program prebere pozitivni celi števili a in b in izpiše njun zmnožek.
# Zmnožek izračuna s pomočjo seštevanja.

# Preberemo obe števili.
a = int(input('Vnesi prvo število: '))
b = int(input('Vnesi drugo število: '))

# Zmnožek a * b izračunamo kot vsoto b a-jev (a + a + ... + a).
zmnozek = 0
for i in range(b):
  zmnozek += a

# Zmnožek izpišemo.
print(zmnozek)

Namig

Pomagaj si z odštevanjem.

Rešitev

# Program prebere pozitivni celi števili a in b in izpiše ostanek pri
# deljenju števila a s številom b. Ostanek izračuna s pomočjo odštevanja.

# Preberemo obe števili.
a = int(input('Vnesi prvo število: '))
b = int(input('Vnesi drugo število: '))

# Ostanek izračunamo s pomočjo odštevanja.
ostanek = a
while ostanek >= b:
  ostanek -= b

# Ostanek izpišemo.
print(ostanek)

Program

Iz izkušenj že vemo, da je program postopek za reševanje problema, zapisan v nekem programskem jeziku. Tako kot problem ima tudi program svoj vhod in izhod: vhod sestavljajo vsi podatki, ki jih program pridobi iz zunanjega sveta (npr. z uporabnikovim vnosom), izhod pa tvorijo podatki, ki jih program zunanjemu svetu sporoči kot svoj rezultat (npr. tako, da jih izpiše na zaslon).

Kdaj lahko rečemo, da program rešuje določen problem? Takrat, ko se program za vsak veljaven vhod problema ustavi in proizvede pričakovani izhod. Pogoj, da se mora program prej ali slej ustaviti (zaključiti svoje izvajanje), se morda zdi samoumeven, kasneje pa bomo videli, da ni čisto tako. Napišimo program za problem MIN3. Možnosti je kar precej, ena od njih pa je sledeča:

Izvedi Počisti 



					

					

						



Naš program prejme vhod tako, da ga uporabnik vnese s pomočjo tipkovnice, dobljeni izhod pa izpiše na zaslon. Seveda pa to ni edina možnost: lahko bi, denimo, brali iz datoteke in vanjo tudi pisali.



Ali naš program resnično rešuje obravnavani problem? Ali se za vsako trojico celih števil ustavi in izpiše pravilen (torej pričakovani) rezultat? Ustavljivost ni vprašljiva: ker program vsebuje samo branje, izpis in pogojne stavke, se bo tako ali drugače vedno ustavil. Kako pa bi preverili pravilnost? Lahko bi, na primer, tvorili tisoč različnih trojic celih števil in za vsako preverili izhod. Vendar pa to ne bi zadoščalo, saj bi morali pravilnost preveriti za vsako možno trojico, teh pa je seveda neskončno. Kako lahko kljub temu pokažemo pravilnost programa, bomo videli nekoliko kasneje.



Vaja





Napiši program, ki rešuje problem množenja dveh pozitivnih celih števil. Da bo naloga zanimivejša, za hip pozabi na pythonov operator *.











Vaja





Napiši program, ki rešuje problem računanja ostanka pri deljenju celih števil a in b, kjer je a ≥ 0 in b > 0. Da bo naloga zanimivejša, za hip pozabi na pythonova operatorja * in /.
